# fix-bd9c652726707f02 — E8T3 `/round` command — discarded `getBalance` result; round state read from ephemeral session

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 ELIM

In `src/handlers/E8T3.ts:24-33`, the `/round` command calls `repository.getBalance(...)` but completely ignores the returned `BalanceResult`. The call has side effects in the Redis repository (creates/ensures the player, migrates legacy data, saves state) but the actual data is thrown away. Then at line 35, the command reads `ctx.session.currentRound` to display round info. This is ephemeral session state set during `/join` — if the bot restarts or the user interacts from a different session, `currentRound` is `undefined` and the command reports "No active round in this chat" even when an active round exists in Redis. E8T3's spec requires "consistency in tracking players" — using session-cached state breaks consistency.

The unused `getBalance` call also means the `/round` command never validates whether the user has a username (the check at line 19–22 prevents the bad path, but `getBalance` is called even after a valid check with no useful result consumed).
